01 / The problem

Turn unstructured commands
into attacker intent.

SSH honeypots capture sequences of shell commands, but manual review does not scale to hundreds of thousands of sessions. The project asked whether machine-learning methods could classify attacker tactics and expose behavioral structure in this noisy, multi-label text.

Sessions were mapped to seven MITRE ATT&CK-derived intents: Persistence, Discovery, Defense Evasion, Execution, Impact, Other, and Harmless. One session can express several intents at once, making this a multi-label classification problem rather than a single mutually exclusive category.

02 / Data

Ninety thousand encoded scripts
before the first model.

The raw dataset required substantial preparation. We decoded 90,026 encoded shell scripts, normalized timestamps, split sessions into commands and keywords, cleaned variables and symbols, and checked missing values. TF-IDF with a 1% minimum document frequency reduced roughly 300,000 candidate features to 90 useful dimensions.

Exploration showed Discovery and Persistence as the dominant intents, each near 200,000 occurrences. Their co-occurrence reached 211,281 sessions; Execution also appeared strongly alongside both. At the other extreme, Impact had only 27 occurrences-an imbalance that later became the clearest model weakness.

03 / Supervised

Strong headline metrics.
Persistent minority-class risk.

Random Forest, SVM, and Logistic Regression established classical baselines. Random Forest and SVM both performed strongly on common attack categories, while grid-search tuning produced only marginal improvement-a sign that the representation and dataset structure mattered more than additional parameter search.

Random Forest · baseline0.996 F1

Weighted precision 0.999 and recall 0.994 on the test set.

SVM · tuned0.9966 F1

RBF kernel, C=100, gamma=scale; approximately 1.48% above baseline.

Neither result erased the imbalance problem. Impact remained difficult because its support was vanishingly small. Weighted averages describe performance on the dominant classes well, but they should not be mistaken for uniform reliability across every intent.

04 / Unsupervised

A good score did not mean
meaningful separation.

K-Means and Gaussian Mixture Models were tuned on the TF-IDF representation. K-Means reached a 0.9490 silhouette score; GMM reached 0.9398. Yet t-SNE visualization and cluster inspection told a less flattering story: a dominant cluster absorbed much of the dataset, terms repeated across groups, and attack behaviors overlapped heavily.

Some communities were interpretable-BusyBox and mount commands suggested IoT-oriented activity, while chmod, wget, and SSH indicated file modification and persistence-but the overall segmentation was not clean enough to treat clusters as distinct tactics.

Metric literacy

The most valuable result was the disagreement between a high silhouette score and weak semantic separation. Validation metrics, visual structure, and domain interpretation all had to be considered together.

05 / Language model

Semantics improved the fit-
with one clear blind spot.

We adapted bert-base-uncased with a custom sigmoid classification head for multi-label prediction. The pretrained layers remained frozen; the head was trained for four epochs using AdamW and binary cross-entropy on a 60/20/20 split.

BERT · weighted test metrics0.9937 F1

Precision 0.9974, recall 0.9927, and ROC-AUC 0.9971.

Weakest class0.78 AUC

Harmless activity remained harder to distinguish because of limited examples and imbalance.

The experiment showed the value of semantic representations for command sequences, but it also reinforced a broader lesson: a sophisticated model does not compensate for missing class support.

06 / Takeaway

The comparison mattered
more than the leaderboard.

Classical supervised models were already highly effective on well-represented intents. Unsupervised methods surfaced broad command communities but struggled to produce semantically distinct attack categories. BERT captured multi-label semantics effectively, while its weakest class exposed the same imbalance found across the study.

The practical next steps are not simply “use a larger model”: introduce domain-aware features and network context, rebalance rare intents, test on broader sources, and evaluate whether results generalize beyond the original June 2019–March 2020 honeypot window.
