01 / Context

Modern observability on
an enterprise kernel.

Vesuvius is my Master's thesis project at Politecnico di Torino, developed with a teammate and in collaboration with Rakuten. The system captures security-relevant process and network events in the kernel, transports them efficiently to userspace, and prepares them for structured detection and analysis.

I own the process-monitoring path: process creation, execution, and termination. The parallel network path covers connect, accept, and bind activity. Together, the two views establish the process-to-socket context needed by a runtime-security system.

02 / Architecture

From a kernel event
to structured evidence.

eBPF hooksShared mapsRing bufferGo readerJSON eventsDetection

The kernel component collects minimal event context at stable tracepoints and LSM hooks. Shared maps maintain process and container state, while a ring buffer carries events to a Go userspace reader. The output is normalized as JSON for downstream rules and MITRE ATT&CK-oriented detection.

Process lifecycle

  • sched_process_fork establishes parent-child lineage.
  • sched_process_exec captures the transition to a new executable image.
  • sched_process_exit closes the lifecycle and supports state cleanup.
03 / Compatibility

The version said 4.18.
The capabilities said otherwise.

The deployment target reports a Linux 4.18 kernel, which initially suggested severe limits: no modern BTF/CO-RE workflow, no ring buffer, and an older eBPF feature set. That assumption was wrong because Rocky Linux carries extensive Red Hat backports.

I built and ran focused probes on the actual target. BTF, CO-RE relocations, BPF LSM, ring buffers, and larger BPF stack usage all passed. The evidence changed the architecture: we could use modern libbpf patterns directly instead of maintaining compatibility workarounds based on the upstream version number alone.

04 / Engineering

Small kernel payloads.
Explicit shared contracts.

Kernel code is kept focused on collection and transport. Common headers define event, task, namespace, and architecture data. Maps provide explicit contracts for task state, event scratch space, and container context; heavier parsing and policy remain in Go userspace.

The project uses feature branches and pull requests to keep two parallel workstreams aligned. Process and network telemetry share event types and state models, making integration decisions visible instead of burying them in hook-specific implementations.

Current framing

Vesuvius is active thesis work. This case study focuses on verified architecture and my process-monitoring contribution; it does not claim production deployment or performance results that have not yet been measured.
